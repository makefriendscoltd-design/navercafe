## 1. 전체 흐름 한눈에 보기

네이버 카페 글쓰기 자동화는 아래 순서로 진행됩니다. 각 단계를 코드 한 블록씩 작성하고, 마지막에 하나로 합쳐서 실행하는 방식입니다.

```
[네이버 로그인] → [카페 이동 + iframe 전환] → [글쓰기 버튼 클릭 + 새 탭 전환]
→ [제목 입력] → [본문 입력] → [사진 업로드] → [장소 추가] → [임시등록]
```

핵심 원리는 간단합니다. 우리가 손으로 하는 모든 동작(클릭, 입력, 스크롤 등)을 Python 코드로 대신하는 것입니다. 코드를 직접 작성할 필요 없이, Cursor AI에게 "이 버튼 클릭해", "여기에 글 입력해"라고 한국어로 말하면 코드를 자동으로 작성해 줍니다.

## 2. 사전 준비 — 필수 프로그램 설치

### 2-1. Python 설치

Python은 자동화 코드를 실행하는 프로그래밍 언어입니다.

**설치 순서:**

1. https://www.python.org/downloads/ 에 접속합니다
2. **Download Python 3.12.x** (또는 최신 안정 버전) 버튼을 클릭합니다
3. 다운로드된 설치 파일을 실행합니다
4. **⚠️ 반드시** 설치 화면 하단의 **"Add Python to PATH"** 체크박스에 체크한 후 "Install Now"를 클릭합니다

> **왜 PATH에 추가해야 하나요?**
> PATH에 추가하지 않으면 나중에 명령어 실행 시 "python을 찾을 수 없습니다"라는 에러가 발생합니다. 체크하지 않고 설치했다면 삭제 후 다시 설치하는 것이 가장 빠릅니다.

**설치 확인 방법:**

1. 키보드에서 `Win + R`을 누르고 `cmd`를 입력하여 명령 프롬프트를 엽니다
2. `python --version`을 입력합니다
3. `Python 3.12.x`와 같이 버전이 표시되면 성공입니다

### 2-2. Chrome 브라우저

최신 버전의 Chrome 브라우저가 설치되어 있어야 합니다. 이미 설치되어 있다면 넘어가세요.

> **2026년 최신:** Selenium 4.x부터는 Selenium Manager가 내장되어 있어, **별도로 ChromeDriver를 다운로드하지 않아도 자동으로 관리**해 줍니다. 예전에는 Chrome 버전에 맞는 ChromeDriver를 일일이 받아야 했지만, 이제는 그 과정이 필요 없습니다.

## 4. Jupyter Notebook 확장 프로그램 설치

### 4-1. 왜 Jupyter를 사용하나요?

일반 `.py` 파일은 실행하면 브라우저가 열렸다가 코드가 끝나면 **자동으로 닫힙니다**. 그래서 코드를 한 줄 수정하고 다시 실행하면, 로그인부터 처음부터 다시 해야 합니다.

100번 수정하면 로그인만 100번 해야 하는 셈입니다. 로그인 한 번에 10초라면, 100번이면 1,000초(약 17분)를 로그인에만 낭비하게 됩니다.

Jupyter Notebook(`.ipynb` 파일)은 코드를 **셀(cell) 단위로 나누어 실행**할 수 있어서, 로그인을 한 번만 하면 브라우저가 **계속 열려 있는 상태**에서 나머지 코드를 하나씩 테스트하고 수정할 수 있습니다. 개발 시간을 압도적으로 절약해 줍니다.

> **비유:** `.py` 파일 = 처음부터 끝까지 한번에 요리하는 것. `.ipynb` 파일 = 재료 손질, 볶기, 양념 등 한 단계씩 확인하며 요리하는 것.

### 4-2. 설치 방법

1. Cursor를 실행합니다
2. 왼쪽 사이드바에서 **Extensions** (네모 4개 아이콘, 또는 `Ctrl + Shift + X`) 클릭
3. 검색창에 **"Jupyter"** 입력
4. **Jupyter** (Microsoft 제작) 확장 프로그램의 **Install** 버튼 클릭

### 4-3. Jupyter 파일 만들기

1. Cursor에서 새 파일 생성 (`Ctrl + N`)
2. 파일 이름을 `카페자동화.ipynb`로 저장 (확장자가 반드시 `.ipynb`)
3. 파일을 열면 일반 텍스트 편집기와 다른 특별한 화면이 나타남
4. 상단 또는 좌측의 **"+ Code"** 버튼을 클릭하면 코드를 입력할 수 있는 **셀**이 생성됨

### 4-4. Jupyter 기본 사용법

| 동작 | 방법 |
|------|------|
| 셀 추가 | `+ Code` 버튼 클릭 |
| 셀 실행 | 셀 왼쪽의 ▶ 버튼 클릭, 또는 `Shift + Enter` |
| 전체 실행 (Run All) | 상단 메뉴의 **Run All** 버튼 |
| 전체 초기화 (Restart) | 상단의 **Restart** 버튼 — 모든 변수·브라우저 초기화 |

> **핵심:** Jupyter에서 셀을 실행하면 그 셀의 변수(예: `driver`)가 메모리에 남아 있어서, 다음 셀에서 이어서 사용할 수 있습니다. 이것이 브라우저를 유지하는 비결입니다.

## 6. Step 1 — 네이버 자동 로그인

### 6-1. Jupyter에서 첫 번째 셀 만들기

`카페자동화.ipynb` 파일을 열고, `+ Code`를 클릭하여 첫 번째 코드 셀을 만듭니다.

### 6-2. 코드

아래 코드를 셀에 입력합니다. 처음부터 직접 칠 필요 없이, Cursor에서 `Ctrl + K`를 누르고 다음과 같이 요청하면 AI가 작성해 줍니다:

```
파이썬 셀레니움으로 네이버 로그인 자동화 코드 작성해 줘.
send_keys 대신 pyperclip으로 클립보드에 복사 후 Ctrl+V 방식으로 해줘.
```

**완성 코드:**

```python
# 네이버 로그인
from selenium import webdriver
from selenium.webdriver.common.keys import Keys
from selenium.webdriver.common.by import By
from selenium.webdriver.common.action_chains import ActionChains
import time
import pyperclip
import pyautogui

id = "여기에_네이버_아이디"
pw = "여기에_네이버_비밀번호"

driver = webdriver.Chrome()

driver.get("https://nid.naver.com/nidlogin.login")
time.sleep(2)

# 아이디 입력 (클립보드 → 붙여넣기 방식)
pyperclip.copy(id)
driver.find_element(By.CSS_SELECTOR, "#id").click()
driver.find_element(By.CSS_SELECTOR, "#id").send_keys(Keys.CONTROL, 'v')
time.sleep(1)

# 비밀번호 입력
pyperclip.copy(pw)
driver.find_element(By.CSS_SELECTOR, "#pw").click()
driver.find_element(By.CSS_SELECTOR, "#pw").send_keys(Keys.CONTROL, 'v')
time.sleep(1)

# 로그인 버튼 클릭
driver.find_element(By.CSS_SELECTOR, "#log\\.login").click()
time.sleep(3)
```

### 6-3. 코드 상세 설명

**줄별로 뜯어보기:**

| 줄 | 코드 | 하는 일 |
|----|------|---------|
| 1~7 | `from selenium import ...` | 필요한 도구들을 불러옴 (한 번만 하면 됨) |
| 9~10 | `id = "..."`, `pw = "..."` | 내 아이디와 비밀번호를 변수에 저장 |
| 12 | `driver = webdriver.Chrome()` | Chrome 브라우저를 자동으로 열기 |
| 14 | `driver.get("https://...")` | 네이버 로그인 페이지로 이동 |
| 17~19 | `pyperclip.copy(id)` → `send_keys(Keys.CONTROL, 'v')` | 아이디를 클립보드에 복사 → 입력란에 Ctrl+V |
| 21~23 | 비밀번호도 같은 방식 | |
| 26 | `#log\\.login` | 로그인 버튼의 CSS 셀렉터 (점을 `\\.`으로 이스케이프) |

**왜 send_keys 대신 Ctrl+V를 사용하나요?**

네이버는 보안이 강력합니다. `send_keys("아이디")`로 직접 글자를 입력하면 "자동 입력 방지" 보안이 작동하여 캡차(CAPTCHA)가 뜨거나 로그인이 차단됩니다. 클립보드 붙여넣기 방식을 쓰면 이를 우회할 수 있습니다.

### 6-4. 실행하기

셀 왼쪽의 ▶ 버튼을 클릭합니다. Chrome이 자동으로 열리고, 네이버에 로그인됩니다.

> **핵심:** Jupyter를 쓰고 있으므로 코드 실행이 끝나도 **브라우저가 닫히지 않습니다!** 로그인 상태가 유지됩니다.

### 6-5. 로그인 셀렉터 보충 설명

`#log\\.login` 이 부분이 좀 특이해 보일 수 있습니다. 네이버 로그인 버튼의 실제 HTML id가 `log.login`인데, CSS 셀렉터에서 점(`.`)은 클래스 구분자로 쓰이기 때문에 `\\.`으로 이스케이프(특수문자 처리)해야 합니다. Cursor AI에게 요소를 Copy element 해서 알려주면 이런 이스케이프도 자동으로 처리해 줍니다.

## 7. Step 2 — 카페 글쓰기 화면 이동

이 단계에서는 **카페 이동 → iframe 전환 → 글쓰기 버튼 클릭 → 새 탭 전환**까지 한꺼번에 처리합니다.

### 7-1. 카페 URL 확인하기

먼저 브라우저에서 수동으로 내 카페에 접속하여, 글을 쓰고 싶은 **게시판 페이지의 URL**을 복사합니다.

예시: `https://cafe.naver.com/카페주소?iframe_url=/ArticleList.nhn%3Fsearch.clubid=카페번호%26search.menuid=게시판번호%26search.boardtype=L`

### 7-2. iframe이란? (핵심 개념)

네이버 카페는 **iframe**이라는 특수한 구조를 사용합니다. 쉽게 말해 하나의 웹페이지 안에 **다른 웹페이지가 액자처럼 끼워져 있는** 구조입니다.

```
┌─────────────────────────────┐
│  네이버 상단 메뉴 (A 사이트)    │
│ ┌─────────────────────────┐ │
│ │                         │ │
│ │  카페 내용 (B 사이트)     │ │  ← 이 부분이 iframe!
│ │  [글쓰기 버튼]           │ │
│ │                         │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

Selenium은 기본적으로 A 사이트만 볼 수 있습니다. B 사이트(iframe) 안의 글쓰기 버튼을 클릭하려면 먼저 **"B 사이트로 전환해줘"**라고 코드로 알려줘야 합니다.

### 7-3. 코드

`+ Code`로 새 셀을 추가하고 아래 코드를 입력합니다:

```python
# 카페 게시판으로 이동
url = "https://cafe.naver.com/yunjadong?iframe_url=/ArticleList.nhn%3Fsearch.clubid=30883250%26search.menuid=1%26search.boardtype=L"
driver.get(url)
time.sleep(2)

# iframe 전환 (카페 내용 영역으로 전환)
driver.switch_to.frame("cafe_main")

# 글쓰기 버튼 클릭
driver.find_element(By.CSS_SELECTOR, "#writeFormBtn").click()
time.sleep(2)

# 새로 생긴 탭으로 이동 (글쓰기를 누르면 새 탭이 열림)
driver.switch_to.window(driver.window_handles[-1])
time.sleep(2)
```

### 7-4. 코드 상세 설명

| 코드 | 하는 일 |
|------|---------|
| `driver.get(url)` | 카페 게시판 페이지로 이동 |
| `driver.switch_to.frame("cafe_main")` | iframe(카페 내용 영역)으로 전환 |
| `find_element(..., "#writeFormBtn").click()` | 글쓰기 버튼 클릭 |
| `driver.switch_to.window(driver.window_handles[-1])` | 마지막에 열린 탭(글쓰기 탭)으로 전환 |

> **⚠️ URL을 반드시 자기 카페 주소로 변경하세요!** 위 URL의 `yunjadong`은 강의자의 카페 주소입니다. `clubid`와 `menuid`도 자기 카페에 맞게 바꿔야 합니다.

### 7-5. iframe 전환이 안 되면?

`driver.switch_to.frame("cafe_main")`에서 오류가 나면, 개발자 도구(F12)에서 iframe 태그를 직접 찾아야 합니다:

1. F12 → 화살표 아이콘 클릭 → 카페 콘텐츠 영역 클릭
2. 코드에서 위로 스크롤하여 `<iframe ...>` 태그를 찾기
3. 해당 iframe 태그를 우클릭 → Copy → Copy element
4. Cursor에서 `Ctrl + K` → 붙여넣기 → **"위 아이프레임으로 전환해 줘"**

## 9. Step 4 — 사진 업로드하기

### 9-1. 핵심 원리

사진 업로드 버튼을 클릭하면 **Windows 파일 탐색기**가 열리는데, 이것은 웹 브라우저가 아니기 때문에 Selenium으로는 직접 제어할 수 없습니다. 대신 `pyautogui`를 사용하여 파일 경로를 붙여넣고 엔터를 누르는 방식으로 우회합니다.

```
[사진 버튼 클릭] → [파일 탐색기 열림] → [경로 붙여넣기] → [엔터(열기)]
```

### 9-2. 코드

```python
# 사진 추가 버튼 클릭
driver.find_element(By.CSS_SELECTOR, ".se-image-toolbar-button").click()
time.sleep(2)

# 이미지 파일 경로를 클립보드에 저장
image_path = r"C:\Users\사용자이름\Downloads\업로드할사진.png"
pyperclip.copy(image_path)

# 파일 탐색기에 경로 붙여넣기 + 열기
pyautogui.hotkey('ctrl', 'v')
time.sleep(1)
pyautogui.press('enter')
time.sleep(3)  # 사진 업로드 완료 대기
```

### 9-3. ⚠️ 파일 경로 작성 시 주의할 점

파일 경로를 작성할 때 두 가지 방법 중 **하나만** 사용하세요. 섞어 쓰면 오류가 납니다.

```python
# ✅ 방법 1: r(raw string) + 역슬래시 하나
image_path = r"C:\Users\Yunjadong\Downloads\사진.png"

# ✅ 방법 2: r 없이 + 역슬래시 두 개
image_path = "C:\\Users\\Yunjadong\\Downloads\\사진.png"

# ❌ 잘못된 예: r + 역슬래시 두 개 (경로에 \\가 그대로 들어감)
image_path = r"C:\\Users\\Yunjadong\\Downloads\\사진.png"
```

> **`r`이란?** raw string의 약자로, 역슬래시(`\`)를 특수문자가 아닌 일반 문자로 처리하라는 의미입니다.

### 9-4. 주의사항

- `pyautogui`는 **화면에 파일 탐색기가 보이는 상태**에서만 작동합니다. 브라우저를 최소화하거나 다른 창을 위에 올려놓으면 안 됩니다.
- 사진 업로드 버튼 클릭 후 파일 탐색기가 열릴 때까지 `time.sleep(2)` 이상 기다려야 합니다.
- 사진 업로드 완료까지도 `time.sleep(3)` 이상 기다려야 다음 동작이 정상 작동합니다.

- **Headless 모드 주의:** `pyautogui`를 사용하는 사진 업로드 단계는 Headless 모드(브라우저 화면 숨김)에서 작동하지 않습니다. 브라우저가 화면에 보여야 합니다.

### 15-2. Chrome 사용자 프로필 로드 (로그인 유지 팁)

이미 로그인된 Chrome 프로필을 불러오면, 매번 로그인할 필요가 없습니다:

```python
from selenium.webdriver.chrome.options import Options

options = Options()
options.add_argument(r"--user-data-dir=C:\Users\사용자이름\AppData\Local\Google\Chrome\User Data")
options.add_argument("--profile-directory=Default")

driver = webdriver.Chrome(options=options)
```

> **주의:** 기존에 열려 있는 Chrome 브라우저를 모두 닫은 후 실행해야 합니다.

### 15-3. 개발자 도구 핵심 정리

| 방식 | 복사 대상 | 장점 | 추천 |
|------|----------|------|------|
| Copy element | HTML 전체 코드 | 정보가 풍부하여 AI가 잘 이해 | 초보자 추천 |
| Copy selector | CSS 선택자만 | 간결함 | 익숙해진 후 추천 |

어떤 방식이든 Cursor AI에게 붙여넣고 **"클릭해"** 세 글자면 충분합니다.

### 15-4. 자동화 윤리

- 스팸성 글 대량 등록 → 카페 강퇴 + 네이버 계정 제재 가능
- 처음에는 **임시등록**으로 내용 검토 후 수동 등록 습관 들이기
- 자동화는 반복 업무 효율화 목적으로 사용하고, 타인에게 피해를 주는 용도로는 사용하지 마세요

## 16. 문제 해결 (트러블슈팅)

### Q1. 브라우저가 열리자마자 바로 닫힙니다

`.py` 파일 대신 `.ipynb`(Jupyter)를 사용하세요. 또는 `.py` 파일 마지막에 다음을 추가하세요:

```python
input("엔터를 눌러 종료하세요...")
```

### Q2. "element not found" 또는 "NoSuchElementException" 오류

두 가지 원인이 가능합니다:

1. **페이지 로딩 미완료** → `time.sleep()` 값을 늘려보세요 (예: 1초 → 3초)
2. **iframe 안의 요소** → `driver.switch_to.frame("cafe_main")`을 먼저 실행했는지 확인하세요

### Q3. 캡차(CAPTCHA)가 나타납니다

- `send_keys("아이디")`로 직접 입력하고 있다면 → `pyperclip` + `Ctrl+V` 방식으로 변경
- 또는 Chrome 사용자 프로필 로드 방식 사용 (15-2 참고)
- 안티캡차 서비스(2Captcha 등) 사용도 가능 — 1,000건당 약 $1

### Q4. 사진 업로드가 안 됩니다

체크리스트:
- [ ] 파일 탐색기가 화면에 보이는 상태인가? (다른 창에 가려지면 안 됨)
- [ ] `time.sleep(2)` 이상 기다린 후 `pyautogui`를 실행하는가?
- [ ] 파일 경로가 정확한가? (`r"C:\..."` 형태로 작성)
- [ ] 파일이 실제로 해당 경로에 존재하는가?

### Q5. 네이버 카페 구조가 변경되어 셀렉터가 작동하지 않습니다

네이버는 주기적으로 CSS 클래스명을 변경합니다. 개발자 도구(F12)에서 최신 요소를 다시 찾아 셀렉터를 업데이트하세요. Cursor AI에게 Copy element한 것을 보여주면 새 셀렉터로 코드를 자동 수정해 줍니다.

### Q6. Cursor 무료 사용량이 부족합니다

무료 플랜은 프리미엄 요청 50회/월입니다. 하루에도 다 쓸 수 있습니다. Pro($20/월)를 추천하며, 가입 시 7일 무료 체험이 제공됩니다. 연간 결제($16/월) 시 20% 할인됩니다.

### Q7. pyautogui가 설치되지 않습니다

Windows에서 `pip install pyautogui` 실행 시 오류가 나면:

```bash
pip install pyautogui --user
```

또는 Cursor 터미널에서:

```bash
python -m pip install pyautogui
```
